# 2026-07-07 — Tagesprotokoll

## PROJ-22 Abholungskalender — Debug-Marathon

### Problem
Feiertage und manuelle Blocker wurden im Abholungskalender nicht angezeigt, obwohl:
- Datenbank hatte korrekte Einträge
- Weekend-Markierung funktionierte
- Mehrere Docker-Rebuilds durchgeführt

### Ursachen & Fixes (chronologisch)

1. **Zeitzone-Bug** (initialer Verdacht)
   - `new Date("2026-01-01")` erzeugt UTC-Mitternacht → verschoben um 1-2h
   - Fix: `parseLocalDate()` mit `new Date(jahr, monat-1, tag)` für lokale Mitternacht
   - Hat NICHT das Hauptproblem gelöst

2. **Statisches Prerendering** (zweiter Verdacht)
   - Seite wurde bei Docker-Build statisch gerendert, nie aktualisiert
   - Fix: `export const dynamic = "force-dynamic"; export const revalidate = 0;`
   - Hat auch NICHT geholfen — Container hatte alten Code (Build-Cache?)

3. **Docker Build-Cache** (dritter Verdacht)
   - `docker-compose build --no-cache` hat nicht den neuen Code aufgenommen
   - Erst nach `docker system prune -f` und komplettem Neu-Aufbau ging es
   - Aber: Code war jetzt im Container, Daten immer noch nicht sichtbar

4. **Traefik Reverse-Proxy Probleme** (vierter Verdacht)
   - Container lief, aber Traefik gab 404 zurück
   - Container-Neustart hat tatsächlich den falschen Container gestartet (alter Image-Tag?)
   - Letztlich: Docker-Labels waren korrekt, aber Routing lief über internes Netzwerk
   - Direkter Zugriff auf Container-IP:3000 funktionierte → Traefik-Problem
   - Nach mehrfachen Neustarts ging es plötzlich

5. **🎯 RLS + SSR — Die wahre Ursache** (final)
   - `getBlockedPeriods()` hat den **Anon-Client** verwendet (RLS aktiv)
   - RLS-Policy: `auth.uid() IS NOT NULL` → bei SSR im Docker-Container keine Cookies verfügbar
   - Supabase verweigerte stillschweigend Zugriff (kein Fehler, leere Ergebnisse)
   - Fix: Service-Role-Client verwenden (wie `initializeHolidays` schon tat)
   - **ERFOLG** — Daten sichtbar!

### Tech-Learnings (dauerhaft wichtig)

- **Admin-Seiten mit Server Actions → IMMER Service-Role-Client**
  - Anon-Client + RLS schlägt bei SSR fehl (keine Cookies)
  - Keine Fehlermeldung, einfach leere Daten — schwer zu debuggen!
- **Docker Build-Cache kann trotz `--no-cache` alt bleiben**
  - `docker system prune -f` war nötig
  - Verifikation: Code im Container prüfen (`docker exec cat ...`)
- **Traefik-Routing kann bei Container-Neustart temporär brechen**
  - Mehrfaches Neustarten hilft manchmal
- **`force-dynamic` + `revalidate = 0` für Admin-Daten notwendig**
  - Sonst statisches Prerendering beim Build

### Zeitaufwand
~3-4 Stunden Debugging für ein Feature, das in 30 Minuten gebaut war. Haupt-Kostenfaktor: mehrere verschachtelte Probleme, die sich überlagert haben.
