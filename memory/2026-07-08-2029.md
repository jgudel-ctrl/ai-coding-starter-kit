## PROJ-23 — Architektur-Zusammenfassung

**Datenbank (5 neue/überarbeitete Tabellen):**
1. **`invoices`** — neu aufsetzen, mit Rechnungsnummer, Datum, Kunde, Beträgen
2. **`invoice_items`** — neu aufsetzen, mit Artikelnummer + `matched_article_id` (Link zur Preisliste)
3. **`price_lists`** — Hersteller, Jahr, Upload-Metadaten
4. **`price_list_items`** — Artikelgruppe, Artikelnummer, Preis, Beschreibung
5. **`invoice_imports`** — Import-Historie (wann, was, wie viele, Fehler)

**Matching:**
- Exact match zuerst (Artikelnummer 1:1)
- Dann normalisiert (Leerzeichen/Bindestriche entfernt)
- Dann fuzzy (Levenshtein ≤ 2)
- Confidence-Wert wird gespeichert (1.0 = perfekt, 0.8 = fuzzy)

**RLS:**
- Alle eingeloggten User können Preislisten lesen
- Nur Admins können schreiben/importieren

**Admin-Bereiche:**
- `/verwaltung/preislisten` — Upload + Übersicht
- `/verwaltung/preislisten/unmatched` — Nicht gefundene Artikelnummern
- `/verwaltung/importe` — Import-Historie

**CSV/Excel:**
- SheetJS für XLSX, eigener Parser für CSV
- Encoding-Erkennung (UTF-8 / ISO-8859-1 / Windows-1252)
- Konfigurierbares Spalten-Mapping (weil Easybill-Spalten variieren können)
